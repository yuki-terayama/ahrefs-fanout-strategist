# ahrefs-fanout-strategist

このスキルは、Ahrefs Brand Radarのクエリファンアウト機能と Claude Code を組み合わせ、AI検索（ChatGPT・Perplexity）における自社のコンテンツ対応状況を可視化し、対策アクションまでを提示します。

## Phase 0: 自社のビジネスモデル調査と競合提案

ユーザーから自社ドメインを受け取り、Web調査でビジネスモデルを把握し、競合候補を最大5社まで提案する。最終決定はユーザーが行う。

## Phase 1: Brand Radar設定の確認

ユーザーがBrand Radar UI上で以下の設定を済ませている前提：
- 自社と競合の登録（URL指定必須、短い英単語は誤検知の元になるので避ける）
- ロケーション：日本
- プラットフォーム：ChatGPTとPerplexity（ファンアウトクエリが取れるのはこの2つのみ）

## Phase 2: ファンアウトクエリ取得

Brand Radar APIから、ChatGPTとPerplexityのファンアウトクエリ一覧を取得する。

## Phase 3: クエリスクリーニング

以下のロジックでクエリを絞り込む：
- 競合名が入っているプロンプトを弾く
- 自社のビジネスモデルと関係ないプロンプトを弾く

## Phase 4: 検索意図の解釈

各ファンアウトクエリから、AIがプロンプトをどう解釈しているか（検索意図）をClaudeに読み解かせる。

## Phase 5: 言及率と引用元トップ10の取得

Brand Radarのデータから、各クエリの直近1週間の言及率と引用元トップ10ページのフルURLを取得する。計算ロジックはAEO-Monitorの方式を参考にする。

## Phase 6: site:検索による既存ページ確認

言及がないクエリに絞り、Google Custom Search APIで `site:自社ドメイン ファンアウトクエリ` を実行し、検索結果1〜3位のURLとタイトルを取得する。

## Phase 7: マッピング表の出力

すべてのデータをCSV形式で出力する。言及率で昇順ソート。

| 列 | 内容 |
|---|---|
| プラットフォーム名 | Brand Radar |
| プロンプト | Brand Radar |
| 直近のファンアウトクエリ | Brand Radar |
| ファンアウトクエリから読み取れる検索意図 | Claude生成 |
| 直近1週間の言及率 | Brand Radar |
| 直近1週間の引用元トップ10フルURL | Brand Radar |
| site:検索1〜3位 URL／タイトル | Google Custom Search API |

## Phase 8: 対策クエリの選択

ユーザーがマッピング表から対策すべきクエリを1件選ぶ。

## Phase 9: 深掘り分析と提言生成

選ばれたクエリに対して以下を実行：
1. 引用元トップ5の本文をWebFetch（失敗時はPlaywrightフォールバック）で取得
2. site:検索結果があれば自社既存ページの本文も取得
3. Claudeが本文ベースで「既存改修」か「新規作成」かを判断
4. ルート別に提言を生成
5. mdファイルに保存（`output/{クエリ}_{YYYYMMDD_HHMMSS}.md`）かつ Claude Code内に直接出力

## 提言フォーマット

### 既存改修ルート
- 対象URL
- 現状サマリ
- 引用元トップ5との内容ギャップ（構造／カバレッジ／データ・事例）
- 追加すべきセクション
- 強化すべき既存セクション
- 削除・整理を検討すべき情報
- 構造化データ・スキーマの推奨
- 公開後の検証ポイント

### 新規作成ルート
- ページタイトル案（3案）
- メタディスクリプション案
- ページタイプ
- 想定URLパス
- ターゲット読者
- 想定文字数
- 構成案（h1〜h3）
- 各セクションで触れるべきポイント
- 引用元から学ぶべき要素（URL＋該当箇所引用）
- 構造化データ・スキーマの推奨
- 公開後の検証ポイント
